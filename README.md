# СТРОЙ УЧАСТОК

## О проекте

Проект сайта для компании СТРОЙ УЧАСТОК, специализирующейся на механоактивированных цементных материалах.

## Как редактировать код?

Есть несколько способов редактирования приложения.

**Использовать предпочитаемый IDE**

Если вы хотите работать локально, используя свой IDE, клонируйте этот репозиторий и отправляйте изменения.

Единственное требование - наличие Node.js & npm - [установка с nvm](https://github.com/nvm-sh/nvm#installing-and-updating)

Выполните следующие шаги:

```sh
# Шаг 1: Клонируйте репозиторий используя Git URL проекта.
git clone <YOUR_GIT_URL>

# Шаг 2: Перейдите в директорию проекта.
cd <YOUR_PROJECT_NAME>

# Шаг 3: Установите необходимые зависимости.
npm i

# Шаг 4: Запустите сервер разработки с автоматической перезагрузкой и мгновенным предпросмотром.
npm run dev
```

**Редактировать файл напрямую в GitHub**

- Перейдите к нужному файлу(ам).
- Нажмите кнопку "Edit" (иконка карандаша) в правом верхнем углу просмотра файла.
- Внесите изменения и зафиксируйте их.

**Использовать GitHub Codespaces**

- Перейдите на главную страницу вашего репозитория.
- Нажмите на кнопку "Code" (зеленая кнопка) в правом верхнем углу.
- Выберите вкладку "Codespaces".
- Нажмите на "New codespace" для запуска новой среды Codespace.
- Редактируйте файлы напрямую в Codespace и зафиксируйте и отправьте изменения после завершения.

## Как развернуть этот проект на GitHub Pages?

### Автоматический деплой (рекомендуется)

1. **Создайте репозиторий на GitHub** (если еще не создан)
   - Перейдите на [GitHub](https://github.com) и создайте новый репозиторий
   - Название репозитория может быть любым (например, `StroySite`)

2. **Загрузите код в репозиторий**
   ```sh
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_USERNAME/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git
   git push -u origin main
   ```

3. **Включите GitHub Pages**
   - Перейдите в Settings → Pages вашего репозитория
   - В разделе "Source" выберите "GitHub Actions"
   - Сохраните изменения

4. **Деплой запустится автоматически**
   - При каждом push в ветку `main` будет автоматически запускаться деплой
   - После успешного деплоя ваш сайт будет доступен по адресу:
     - Если репозиторий называется `username.github.io`: `https://username.github.io`
     - Иначе: `https://username.github.io/repository-name/`

### Ручной деплой

Если хотите задеплоить вручную:

```sh
# Соберите проект
npm run build

# Если репозиторий НЕ называется username.github.io, установите base path
# Например, для репозитория "StroySite":
VITE_BASE=/StroySite/ npm run build
```

Затем загрузите содержимое папки `dist` в ветку `gh-pages` вашего репозитория.
